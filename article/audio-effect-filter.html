Effect - Filter & EQ

2026-05-09

從《 Designing Audio Effect Plug-Ins in C++ 》了解一下基礎特性

Filter

濾波就是對某些頻率增益、對某些頻率衰減，差別在於用什麼工具來決定那個形狀。

雙線性 z 轉換（BZT）

類比濾波器的設計方法已經成熟幾十年，BZT 就是模擬這些既有類比設計。類比用的是 s 平面，數位用的是 z 平面，核心代換就是把類比轉換函式 H(s) 裡所有的 s 換成 (z-1)/(z+1) 得到數位版的 H(z)。兩者的對應關係是：

  • s 平面的虛數軸（jω 軸）← 對應 → z 平面的單位圓
  • s- 平面左半邊（穩定區域） ← 對應 → z 平面單位圓內部
  • 類比的 Laplace 轉換 ← 對應 → 數位的 z 轉換

BZT 有一個副作用：類比的線性頻率軸在轉換後會被「壓縮」，越靠近 Nyquist 的頻率壓縮越嚴重，導致截止頻率跑掉。解決方法是事前補償（pre-warp），先把目標頻率「拉開」ωa = tan(ωd / 2)其中 ωd 是你想要的數位截止頻率，ωa 是代入 H(s) 的類比截止頻率。BZT 壓縮之後，ωa 剛好會落回你原本想要的 ωd 位置。

z 平面與單位圓

數位音訊就是一串數字樣本x(0), x(1), x(2)...，而每個樣本比前一個晚了一個樣本週期。z⁻¹ 代表「往前延遲一格」x(n) × z⁻¹ = x(n-1)，取的是上一個樣本的值。

當我們問「這個濾波器對 1kHz 做了什麼？」，就是把 z 替換成代表 1kHz 的複數，計算輸出／輸入的比值。而所有音訊頻率對應的 z 值，會全部落在一個半徑為 1 的圓上(數學上的必然)，這就是單位圓（Unit Circle）。

  • 極點 極點讓靠近它的頻率增益變大，就像磁鐵把單位圓上附近那段頻率的響應往上拉。

    • 極點越靠近單位圓 → 峰值越尖銳（Q 越高）
    • 極點碰到單位圓 → 無限增益 = 振盪器
    • 極點跑到單位圓外 → 系統不穩定（爆炸）

  • 零點

    零點讓靠近它的頻率增益變小，就像黑洞吸走了那個頻率的能量，使輸出趨近於零。

    • 零點越靠近單位圓 → 凹陷越深
    • 零點在單位圓上 → 完全消去該頻率（凹陷至 −∞）
    • 零點可放在任意位置，不影響系統穩定性

所以設計數位濾波器，本質上就是在這張頻率地圖上決定點要放哪裡：

  • 想讓 1kHz 有峰值 → 把極點放在圓內 1kHz 角度附近
  • 想完全消除 60Hz 電源嗡嗡聲 → 把零點直接放在圓上 60Hz 的位置（碰到圓 = 完全歸零）

IIR（Infinite Impulse Response）

IIR 濾波器具有 feedback，也就是輸出會回頭參與下一次運算，因此計算量低、效率高，很適合即時音訊處理。不過它的相位通常是非線性的，設計不當時也可能產生不穩定，嚴重時甚至會讓輸出爆掉。判斷穩定性的方式，是檢查極點是否位於 z 平面的單位圓內。

最常見的 IIR 實作單元是 Biquad（二階濾波器），幾乎所有常見 EQ 都可以由它堆疊或變形而來，例如 LPF、HPF、BPF、Shelf 與 Peak EQ。最常見的係數設計參考是 RBJ Cookbook。

y(n) = a0·x(n) + a1·x(n-1) + a2·x(n-2)
                - b1·y(n-1) - b2·y(n-2)

z 平面

直接決定極點和零點要放在 z 平面的哪個位置，再反推係數。適合目標明確的場景，例如「在 2kHz 放一個共鳴峰」或「在 60Hz 挖一個凹陷」， Resonator 和 APF 都是這種方式。

BZT

使用成熟的類比濾波器設計作為原型，大部分的 Biquad EQ 係數都是這樣來的。

  1. 取正規化的類比轉換函式 H(s)（截止頻率為 1 rad/sec）
  2. Pre-warp：用 ωa = tan(ωd / 2) 把目標截止頻率補償好，避免 BZT 造成的頻率壓縮偏移
  3. 把 H(s) 的截止頻率從 1 縮放到 ωa（把 s 換成 s / ωa）
  4. 套用 BZT：把所有 s 換成 (z-1)/(z+1)，得到 H(z)
  5. 整理成 Biquad 格式，提取 a0, a1, a2, b1, b2

常見的類比原型包括：

  • Butterworth：頻率響應最平坦，過渡帶較寬
  • Chebyshev：過渡帶較陡，但會出現 ripple
  • Elliptic：過渡帶最陡，但通帶與阻帶兩側都會有 ripple

Q 值

對二階低通而言，Butterworth 的臨界點是 Q = 0.707。當 Q ≤ 0.707 時，不會出現 peaking，fc 對應的是 -3 dB 頻率；當 Q > 0.707 時，響應開始在截止附近產生凸起，這時 fc 更接近 peak 所在位置，而不再等同於 -3 dB 點。

Modified Biquad

音訊領域常用 Modified Biquad，在標準 Biquad 上額外加入 c0 與 d0 兩個係數，用來控制 wet/dry mix，也就是把原始輸入直接混回濾波輸出。這種結構常見於 Shelving Filter、Parametric EQ 與 Graphic EQ，因為這些音訊 EQ 不只是單純濾波，還需要保留部分原始訊號來塑造更自然、可調的頻率響應。

FIR（Finite Impulse Response）

FIR 濾波器只有 feed-forward 路徑，沒有 feedback，因此絕對穩定，輸出不可能爆炸。它最重要的特性是係數本身就是衝激響應（IR）—— 把一個脈衝打進去，讀出來的那串輸出數字，就等於所有係數 h(0), h(1)…h(N)。

y(n) = h(0)·x(n) + h(1)·x(n-1) + h(2)·x(n-2) + ... + h(N)·x(n-N)

這個運算在時域叫做卷積（Convolution）。訊號 x(n) 每個樣本週期往右滑動一格，和凍結在緩衝區裡的 IR 係數逐一相乘再加總，就得到輸出。

FIR 的另一個獨特優勢是可以做到線性相位：只要係數左右對稱（h(k) = h(N-k)），所有頻率的延遲時間就完全一致，不會造成任何相位失真。IIR 做不到這點，缺點是需要大量係數才能達到陡峭的響應，幾十到幾千個都有可能，計算量遠高於 IIR。

z 平面

FIR 只有零點，沒有極點。零點決定了頻率響應的凹陷位置，通帶和阻帶之間的過渡就是靠這些零點的分佈來塑造的。

直接截取 IR

若你已知某個系統的 IR（例如真實空間的殘響、或從 IIR 濾波器截出來的響應），直接把它當係數用。這是最直觀的方式，「擁有什麼系統的 IR，就能複製那個系統的聲音」。

頻率取樣法

在頻率軸上直接畫出想要的響應形狀，在等間距位置取樣幾個點，做反 DCT（離散餘弦轉換）得到 IR 係數。由於係數自動對稱，保證線性相位。可以設計任意形狀，不限於 LPF/HPF 等傳統類型。缺點是取樣點之間的響應無法精確控制，容易在通帶和阻帶出現漣波（ripple）。

最佳化法

指定通帶漣波容許量、阻帶衰減量，以及濾波器階數，用 Remez 交換算法反覆迭代，把零點移到最佳位置，最終採樣結果用頻率取樣法轉成 IR 係數。

這是實務上最常用的 FIR 設計方式，可以做出等漣波（equiripple）設計——通帶和阻帶的誤差均等分配，在給定階數下達到最佳效果。限制是演算法不保證收斂，規格太嚴苛時可能找不到解。

Modulated Filter

把濾波器的某個參數接上一個會動的控制訊號，讓音色隨時間變化。類比電路裡，濾波器的截止頻率由電容和電阻決定。如果換成一顆可變電阻，再用電壓去控制它，截止頻率就會跟著電壓走。

Mod Filter

LFO 控制截止頻率。LFO 輸出的波形 → 直接控制濾波器的 fc 和 Q 讓音色產生週期性變化。數位上由 LFO 輸出一個 -1 到 +1 的波形值，換算成 fc 的範圍（比如 100Hz 到 5kHz），每個 sample 都重新算一次濾波器係數再過濾。LFO 是機械式的、規律的，所以聽起來有點「掃頻」的機械感。

  • Part I：LFO → 只調 fc
  • Part II：兩個 LFO 分別調 fc 和 Q
  • Part III：雙 LFO 差 90° 相位（正交）→ 左右聲道不同步，聲音在空間飄移

Envelope Follower

偵測輸入訊號的 RMS 音量，音量大就把 fc 推高，音量小就讓 fc 回到最低值。效果會跟著演奏的強弱呼吸，這就是 Auto-Wah 的工作原理。

  • 超過 Threshold 才觸發，低於時 fc = 最小值
  • Attack / Release 控制包絡的反應速度
  • 可設定「音量↑→fc↑」或「音量↑→fc↓」

Phaser

不改變音量頻率響應，而是讓訊號通過 6 個全通濾波器（APF），這些濾波器只偏移相位不衰減幅度。最後把相移後的訊號與乾訊號混合，某些頻率因為相位相消出現凹陷點（notch）。LFO 掃動這些凹陷點的位置，就是 Phaser 那種飄移的音色。它和 Flanger 的差別在於：Flanger 的 notch 是等間距的諧波關係，Phaser 不是，所以兩者聽感完全不同。

  • APF 不改變音量，只改變相位，各級 fc 範圍不同（16Hz ~ 26kHz）
  • 與乾訊號混合後，某些頻率因相位抵消出現「notch」（凹陷點）
  • Depth = 50% 混合時效果最明顯；Feedback 增加「Intensity」強度
