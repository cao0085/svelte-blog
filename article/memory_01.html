2026-06-05

認識半導體記憶體

接續 MCU 那條線：MCU 第一個瓶頸就是「記憶體不夠大」。這篇把記憶體本身拆開——它是另一條獨立、且資本與週期性都更重的產業鏈。

記憶體最關鍵的分法是 揮發性（volatile） 與 非揮發性（non-volatile）：斷電就忘的是工作記憶，斷電還記得的是儲存。

記憶體
├── 揮發性 volatile（斷電即失，當「工作記憶」）
│   ├── SRAM：快、貴、不需刷新，常做 cache / MCU 內部 RAM
│   └── DRAM：密度高、需定時刷新，當主記憶體
└── 非揮發性 non-volatile（斷電保留，當「儲存」）
    ├── NOR Flash：可隨機定址、適合放程式碼（code）
    ├── NAND Flash：密度高、便宜、適合放大量資料（SSD / eMMC / UFS）
    └── 新興：MRAM、ReRAM、PCM 等

三大主角

各自的定位、為什麼存在、用在哪裡：

  • DRAM：當主記憶體（main memory），密度高、單位成本低，但斷電即失、且要持續刷新。
  • NAND Flash：當大量儲存（SSD / eMMC / UFS），密度最高、最便宜，斷電保留但有寫入壽命。
  • NOR Flash / SRAM：和 MCU 直接相關——NOR 可隨機定址、適合放程式碼（execute in place），SRAM 快又免刷新、常做 cache 或 MCU 內部 RAM。

DRAM（動態隨機存取記憶體）

每個位元用「一顆電晶體 + 一顆電容」（1T1C）存一個電荷：有電是 1、沒電是 0。但電容會漏電，必須每隔幾毫秒就把資料讀出再寫回一次，這個動作叫 刷新（refresh），「動態（Dynamic）」就是指這件事。代價是斷電即失、且刷新本身要耗電；換來的是結構極簡，所以密度高、單位成本遠低於 SRAM，這就是它被選來當主記憶體的原因——容量大、夠便宜，速度雖不如 SRAM 但可接受。

製程上 DRAM 和 MCU 走相反路線：它一直在追先進製程，但卡點不在邏輯閘而在那顆電容——微縮後電容太小、存不住電荷又更容易漏，物理逼近極限，所以 10nm 世代不再標數字，改用 1x → 1y → 1z → 1α → 1β → 1γ 這種代號往下擠。

規格高度標準化，由 JEDEC 制定，依應用分成幾條主線：

  • DDR：PC 與伺服器主記憶體，目前主流 DDR5。
  • LPDDR：行動裝置用，主打低功耗。
  • GDDR：顯卡用，衝頻寬。
  • HBM：把多顆 DRAM 用 TSV 垂直堆疊、貼在運算晶片旁，超高頻寬，是 AI / GPU 時代最熱的一條（後面 2025–2026 焦點再展開）。

產業結構是典型寡占——全球 DRAM 幾乎被 三星、SK 海力士、美光 三家吃下。進入門檻極高（先進製程 + 鉅額資本 + 良率 know-how），新玩家極難切入；也因為產品近乎商品（commodity）、規格統一，廠商拚的是成本與良率，供需一旦錯配，價格就大漲大跌，形成劇烈的「記憶體景氣循環」。

補充：對照 mcu-02——MCU 多是 fabless + 成熟製程 + 利基分散；DRAM 正好相反，IDM 自有晶圓廠、追先進製程、高度集中。同樣是半導體，商業邏輯完全不同。

NAND Flash（大量儲存）

非揮發性，斷電資料還在，靠「浮閘（floating gate）／電荷捕捉層」把電荷關起來存資料。它的命名來自電路結構——記憶單元像 NAND 邏輯閘那樣串聯在一起，省下接線面積，所以密度最高、單位成本最低；代價是不能像 NOR 那樣隨機定址，讀寫以「頁（page）」為單位、抹除要以更大的「區塊（block）」為單位，而且每個單元能抹寫的次數有限（P/E cycle 壽命），要靠控制器做平均抹寫（wear leveling）。

NAND 的兩條進化主軸要分清楚：

  • 一格存幾個位元：SLC（1）→ MLC（2）→ TLC（3）→ QLC（4）。塞越多位元越便宜，但越慢、越不耐寫。
  • 從 2D 到 3D：平面微縮到極限後改成「往上疊層」，現在主流堆到 200 層以上，拚的是層數而不是線寬。

用在 SSD、手機的 eMMC / UFS、記憶卡、隨身碟。玩家是三星、SK 海力士（含併入的 Solidigm）、美光、鎧俠（Kioxia）、Sandisk（原 WD 記憶體部門），同樣是寡占。NAND 需要一顆 主控（controller） 才能用，這給了台廠群聯（Phison）、慧榮（Silicon Motion）切入的空間——他們不做顆粒、做控制晶片與韌體。

NOR Flash 與 SRAM（與 MCU 最近的兩個）

這兩個容量小、單價高，但反而是和 MCU 綁最緊、也是台廠的機會點：

  • NOR Flash：記憶單元並聯，可隨機定址、讀取快，支援就地執行（XIP, execute in place），所以適合放程式碼與韌體——MCU 外掛的 code flash、BIOS、車用都是它。容量做不大，但穩定可靠。台廠旺宏（Macronix）、華邦（Winbond）是全球要角。
  • SRAM：一個位元要用六顆電晶體（6T）鎖住狀態，不需刷新、速度最快，但最不密、最貴。通常不做成大容量獨立晶片，而是整合進 CPU cache、MCU 內部 RAM——也就是你在 mcu-01 看到的「RAM 不夠大」那個 RAM。

為什麼記憶體是「重資本 + 強週期」

和 MCU（成熟製程、利基、fabless）對照，記憶體幾乎每一點都走相反路線，這也是它自成一條產業鏈的原因：

  • 追先進製程：DRAM、NAND 都吃先進製程與鉅額設備投資，和 MCU 卡在成熟製程相反，蓋一座廠動輒百億美元。
  • 高度標準化、近商品（commodity）：規格由 JEDEC 統一，產品差異小 → 拚的是成本與良率 → 沒有 MCU 那種靠生態工具鏈黏住客戶的護城河。
  • 寡占但週期劇烈：玩家極少、門檻極高，可是只要供需一錯配，價格就暴漲暴跌，形成有名的「記憶體景氣循環」，廠商獲利大起大落。
  • IDM 為主：幾乎都自己有晶圓廠（三星、SK 海力士、美光），不像 MCU 可以 fabless 輕資產起家。

2025–2026 的焦點

和現在的產業熱點對上，記憶體最受矚目的三條線：

  • HBM（高頻寬記憶體）：把多顆 DRAM 用矽穿孔（TSV）垂直堆疊、貼在 GPU / AI 加速器旁邊，餵得動龐大算力。AI 浪潮讓 HBM 變成 DRAM 三強的兵家必爭，SK 海力士在這塊一度領先，是目前記憶體最賺、最缺的產品。
  • 先進封裝把記憶體拉進來：HBM 要靠 CoWoS 這類 2.5D / 3D 封裝和運算晶片整合——這條線直接接到你之後想寫的「封測 / 先進封裝」，記憶體和邏輯晶片不再是兩件事。
  • 地緣與供應鏈風險：呼應 mcu-02 的成本與國際因素——記憶體同樣有出口管制、產能集中（韓國）、中國長江存儲（YMTC）／長鑫（CXMT）追趕的變數，HBM 與先進製程記憶體是管制重點。
